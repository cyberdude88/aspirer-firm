# Aspirer Firm

Next.js (App Router, TS, Tailwind) site with:

- **Supabase** — auth + bookings persistence (`@supabase/ssr`, service-role for server writes)
- **Stripe** — Checkout + webhook → creates the calendar event
- **Google OAuth** — visitor sign-in (NextAuth, Google provider, calendar.events scope)
- **Google Calendar** — owner-account refresh token; server lists free/busy and inserts events

Visual baseline from the original GoDaddy site lives in `design-reference/` (scraped 2026-05-12, GoDaddy analytics/CDN cruft stripped).

## Quick start

```bash
cp .env.local.example .env.local
# fill in real values, then
npm run dev
```

Open <http://localhost:3000>.

## Routes

| Path                          | Purpose                                                 |
| ----------------------------- | ------------------------------------------------------- |
| `/`                           | Home                                                    |
| `/booking`                    | List of bookable services                               |
| `/booking/[service]`          | Show available slots → Stripe Checkout                  |
| `/api/availability?slug=...`  | Free/busy via owner's Google Calendar                   |
| `/api/stripe/checkout`        | Create Stripe Checkout session                          |
| `/api/stripe/webhook`         | Stripe → create calendar event + persist to Supabase    |
| `/api/auth/[...nextauth]`     | NextAuth (Google) for visitor sign-in                   |

## One-time setup checklist

1. **Google Cloud** — create OAuth client (Web), redirect URI `http://localhost:3000/api/auth/callback/google`. Enable Google Calendar API. Save the client ID/secret.
2. **Owner refresh token** — run the OAuth consent flow once for the business Gmail account with `scope=https://www.googleapis.com/auth/calendar.events&access_type=offline&prompt=consent`. Save the `refresh_token` into `GOOGLE_OWNER_REFRESH_TOKEN`.
3. **Supabase** — create project, add `bookings` table:
   ```sql
   create table bookings (
     id bigint generated by default as identity primary key,
     created_at timestamptz default now(),
     slug text not null,
     slot timestamptz not null,
     customer_email text not null,
     stripe_session_id text not null
   );
   ```
4. **Stripe** — create one Product + Price per service, paste price IDs into `STRIPE_PRICE_*`. Forward a local webhook with `stripe listen --forward-to localhost:3000/api/stripe/webhook`.

## Layout

```
src/
  app/
    page.tsx                    home
    booking/page.tsx            service list
    booking/[service]/page.tsx  slot picker + checkout
    api/
      availability/route.ts     GET ?slug=
      stripe/checkout/route.ts  POST { slug, slot }
      stripe/webhook/route.ts   POST (Stripe events)
      auth/[...nextauth]/       NextAuth Google provider
  lib/
    supabase.ts   browser / server / admin clients
    stripe.ts     server Stripe SDK + slug→priceId map
    google.ts     Calendar client, free/busy, event create
    services.ts   single source of truth for service slugs
design-reference/  scraped HTML from the existing GoDaddy site (visual only)
```
